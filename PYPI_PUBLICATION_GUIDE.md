# 📦 Publishing QuantLib Pro to PyPI\n\nThis guide will walk you through publishing your QuantLib Pro package to the Python Package Index (PyPI).\n\n## Prerequisites ✅\n\n### 1. PyPI Accounts\nCreate accounts on:\n- **Test PyPI**: https://test.pypi.org/account/register/ (for testing)\n- **Production PyPI**: https://pypi.org/account/register/ (for final release)\n\n### 2. Generate API Tokens\nFor secure authentication:\n\n**Test PyPI**:\n1. Go to https://test.pypi.org/manage/account/token/\n2. Create new token with scope \"Entire account\"\n3. Save the token (starts with `pypi-`)\n\n**Production PyPI**:\n1. Go to https://pypi.org/manage/account/token/\n2. Create new token with scope \"Entire account\"\n3. Save the token securely\n\n### 3. Install Build Tools\n```bash\npip install --upgrade build twine\n```\n\n## Package Preparation 📋\n\n### 1. Create Essential Files\n\n#### MANIFEST.in\nCreate file to include additional files in the distribution:\n\n```plaintext\ninclude README.md\ninclude LICENSE\ninclude pyproject.toml\ninclude requirements.txt\nrecursive-include docs *\nrecursive-include quantlib_pro/data *.json *.csv\nrecursive-exclude * __pycache__\nrecursive-exclude * *.py[co]\n```\n\n#### LICENSE\nMIT License (already specified in pyproject.toml):\n\n```\nMIT License\n\nCopyright (c) 2024 QuantLib Pro Team\n\nPermission is hereby granted, free of charge, to any person obtaining a copy...\n```\n\n#### Long Description README\nEnsure your README.md is comprehensive for PyPI display.\n\n### 2. Verify Package Structure\nYour current structure is excellent:\n```\nquantlib-pro/\n├── quantlib_pro/          # Main package\n├── quantlib_api/          # API package  \n├── quantlib_cli/          # CLI package\n├── docs/                  # Scientific documentation\n├── tests/                 # Test suite\n├── pyproject.toml         # Modern Python packaging\n├── README.md              # Package description\n└── setup.py               # Fallback setup\n```\n\n## Building and Testing 🔨\n\n### 1. Clean Previous Builds\n```bash\n# Remove old build artifacts\nRemove-Item -Recurse -Force dist, build, *.egg-info -ErrorAction SilentlyContinue\n```\n\n### 2. Build the Package\n```bash\n# Build source and wheel distributions\npython -m build\n```\n\nThis creates:\n- `dist/quantlib-pro-1.0.0.tar.gz` (source distribution)\n- `dist/quantlib_pro-1.0.0-py3-none-any.whl` (wheel distribution)\n\n### 3. Verify the Build\n```bash\n# Check distribution contents\ntwine check dist/*\n```\n\n## Publishing Process 🚀\n\n### Step 1: Upload to Test PyPI\nAlways test first!\n\n```bash\n# Upload to test PyPI\ntwine upload --repository testpypi dist/*\n```\n\n**Configure credentials** when prompted:\n- Username: `__token__`\n- Password: Your Test PyPI API token\n\n### Step 2: Test Installation from Test PyPI\n```bash\n# Test installation\npip install --index-url https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple/ quantlib-pro\n```\n\n### Step 3: Test the Package\n```python\n# Test basic functionality\nfrom quantlib_pro import QuantLibSDK\n\nsdk = QuantLibSDK()\nprint(\"QuantLib Pro successfully installed!\")\nprint(f\"Available modules: {dir(sdk)}\")\n```\n\n### Step 4: Upload to Production PyPI\nOnce testing passes:\n\n```bash\n# Upload to production PyPI\ntwine upload dist/*\n```\n\n**Use your Production PyPI token**:\n- Username: `__token__`\n- Password: Your Production PyPI API token\n\n## Post-Publication Setup 🎯\n\n### 1. Configure PyPI Project Settings\n\n**On PyPI.org after first upload**:\n1. Go to https://pypi.org/project/quantlib-pro/\n2. Click \"Manage\" → \"Settings\"\n3. Set up project description, links, maintainers\n\n### 2. Set Up Automated Publishing\n\n**GitHub Actions for CI/CD** (`.github/workflows/publish.yml`):\n\n```yaml\nname: Publish to PyPI\n\non:\n  release:\n    types: [published]\n\njobs:\n  build-and-publish:\n    runs-on: ubuntu-latest\n    steps:\n    - uses: actions/checkout@v3\n    \n    - name: Set up Python\n      uses: actions/setup-python@v4\n      with:\n        python-version: '3.11'\n    \n    - name: Install build tools\n      run: pip install build twine\n    \n    - name: Build package\n      run: python -m build\n    \n    - name: Publish to PyPI\n      env:\n        TWINE_USERNAME: __token__\n        TWINE_PASSWORD: ${{ secrets.PYPI_API_TOKEN }}\n      run: twine upload dist/*\n```\n\n### 3. Badge for README\nAdd PyPI badge to your README.md:\n\n```markdown\n[![PyPI version](https://badge.fury.io/py/quantlib-pro.svg)](https://badge.fury.io/py/quantlib-pro)\n[![Downloads](https://pepy.tech/badge/quantlib-pro)](https://pepy.tech/project/quantlib-pro)\n```\n\n## Version Management 📊\n\n### Semantic Versioning\nFollow semantic versioning (semver.org):\n- `1.0.0` → `1.0.1` (bug fixes)\n- `1.0.1` → `1.1.0` (new features, backward compatible)\n- `1.1.0` → `2.0.0` (breaking changes)\n\n### Releasing New Versions\n\n1. **Update version in pyproject.toml**:\n   ```toml\n   version = \"1.0.1\"\n   ```\n\n2. **Create changelog entry**\n\n3. **Build and upload**:\n   ```bash\n   python -m build\n   twine upload dist/*\n   ```\n\n4. **Tag release on GitHub**:\n   ```bash\n   git tag v1.0.1\n   git push origin v1.0.1\n   ```\n\n## Installation Commands for Users 💻\n\nOnce published, users can install:\n\n```bash\n# Basic installation\npip install quantlib-pro\n\n# SDK only (minimal dependencies)\npip install quantlib-pro[sdk]\n\n# Full platform with UI\npip install quantlib-pro[full]\n\n# All features\npip install quantlib-pro[all]\n\n# Development version\npip install quantlib-pro[dev]\n```\n\n## Troubleshooting 🔧\n\n### Common Issues\n\n**1. \"Package already exists\"**\n- Version already published\n- Increment version number\n\n**2. \"Invalid credentials\"**\n- Check API token format\n- Ensure token has correct scope\n\n**3. \"File size too large\"**\n- Remove unnecessary files\n- Use `.pypiignore` or improve MANIFEST.in\n\n**4. \"Invalid metadata\"**\n- Validate pyproject.toml\n- Check all required fields\n\n### Security Best Practices\n\n1. **Never commit API tokens**\n2. **Use environment variables**:\n   ```bash\n   export TWINE_USERNAME=__token__\n   export TWINE_PASSWORD=your-api-token\n   ```\n3. **Enable 2FA on PyPI accounts**\n4. **Use project-scoped tokens when possible**\n\n## Success Metrics 📈\n\nAfter publication, monitor:\n- **Download statistics**: https://pypistats.org/packages/quantlib-pro\n- **PyPI
